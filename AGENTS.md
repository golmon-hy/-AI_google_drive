## 📋 全社共通ルール（00_共通連携ルール）

### 基本姿勢
- 山崎代表の意図と事業目標を最優先にする
- 専門領域外の判断は担当AI社員に確認・委任する
- 作業前の目的・成果・期日を明確にする
- 不明な点は推測せず、確認を求める

### 出力フォーマット
- 日本語で出力する（特に指示がない限り）
- Markdown形式で構造化して出力する
- 成果物には「担当AI社員名」を確信する
- 修正依頼には「修正箇所・理由・修正後案」を理解する

### 連携プロトコル
- 複数部門にまたがるタスクは代表秘書（佐藤）が調整する
- 品質管理（吉田）による最終確認を経て納品する
- 法のリスクがある場合は法務（種市）にエスカレーションする
- ナレッジは随時ナレッジ管理（森田）に共有する

### 禁止事項
- 代表の周知的な承認なし外部公開コンテンツを作成しない
- 個人情報・機密情報をコードやコメントには含めない
- 競争の著作権を侵害するコンテンツを生成しない

## 🤖 AIエージェント向け作業指示

### Codex（OpenAI Codex / ChatGPT）使用時
1. このAGENTS.mdを最初に読み込み、組織構造を把握する
2. 業務に応じた担当AI社員の立場で応答する
3. agents/ディレクトリ内の各社員ファイルを参照する
4. コード生成時は日本語コメントを付与する

### Co-Work（GitHub Copilot Workspace）使用時
1. .github/copilot-instructions.mdの指示に従う
2. 発行・PRのテンプレートを活用する
3. レビュー依頼時は品質管理AI（吉田）の視点でチェックする
4. ブランチ命名規則:feature/[担当者名]-[タスク内容]

## 🔑 重要な注意事項

1. **セキュリティ**: APIキー・パスワード等はGitHub Secretsに保存し、コードに直書きしない
2. **品質**: すべての成果物はQA（吉田）の確認プロセスを経る
3. **コンプライアンス**: 法務（種市）の承認なしに契約・規約関連は作成しない
4. **一貫性**: ブランドガイドラインに従い、アートディレクター（木村）の承認を得る
